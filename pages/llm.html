LLMs & Foundation
Models.

The engine underneath everything. Understanding what LLMs actually are — and aren't — is the difference between sounding fluent and sounding like you memorized a glossary.

What an LLM actually is

A Large Language Model is a mathematical system trained to predict what word (technically, what token) comes next in a sequence. That's it. The remarkable thing is that doing this at massive scale — billions of parameters, trillions of words of training data — produces a system that appears to reason, summarize, translate, code, and converse.

It doesn't "know" things the way you know things. It has learned statistical patterns about how language works, and those patterns encode an enormous amount of implicit knowledge about the world. When it tells you the capital of France, it isn't looking it up — it's reproducing a pattern it saw countless times in training.

The key mental model Think of an LLM as an incredibly well-read person who has absorbed everything ever written — but who has no memory between conversations, no ability to verify facts in real time, and no direct connection to the world outside of text. Powerful but bounded.

Key concepts to own

Context window

The model's working memory — how much text it can see at once. Claude: ~200K tokens (~150,000 words). GPT-4o: ~128K. Bigger = you can feed more documents, more history, more context in one call.

Temperature

A 0–1 dial controlling randomness. 0 = deterministic and precise (data extraction). 1 = creative and varied (copywriting). Most production systems run 0.2–0.7.

Inference vs. training

Training = building the model (millions of dollars, done by Anthropic/OpenAI). Inference = running the model to get a response (fractions of a cent per API call). You only ever pay for inference.

Knowledge cutoff

The date training data ended. The model has no built-in knowledge after that date. Fix: RAG or web search access.

Multimodal

Models that handle more than text. Claude and GPT-4o can read images. For small business: receipt scanning, product photo analysis, screenshot reading — all possible.

What LLMs are bad at

Real-time information
Training data has a cutoff. Fix: connect to web search or live data via RAG.
Precise arithmetic
LLMs are not calculators — they can get math wrong confidently. Fix: use function calling to run actual code.
Memory between sessions
No memory by default. Everything it "knows" is in the current context window. Fix: store state externally and inject it.
Verifying its own outputs
It doesn't know when it's hallucinating. Fix: ground it to verified sources, add human review for high-stakes outputs.
Monday tip If someone proposes using an LLM for any of these without a mitigation — "How are we handling real-time data?" or "What's our verification step?" — you immediately sound like someone who's built in production.
