MCP &
Connectors.

The bridge between AI and the real world. Without this layer, an LLM can only talk. With it, an LLM can act.

What MCP is

MCP stands for Model Context Protocol — an open standard created by Anthropic that lets AI models connect to external tools, data sources, and services in a standardized way. Think of it as USB for AI: instead of every tool needing a custom integration, any MCP-compatible tool can plug into any MCP-compatible AI.

Before MCP, connecting Claude to your calendar required custom code written specifically for that integration. With MCP, if Google Calendar publishes an MCP server, any AI that supports MCP can use it — with no custom code needed.

Why this is a big deal MCP is rapidly becoming an industry standard, not just an Anthropic thing. OpenAI, Google, and most major AI tooling companies are adopting it. When your meeting talks about "connectors," MCP is the infrastructure underneath — and knowing this puts you ahead of most people in the room.

What tools connect via MCP

Databases
The AI can query your SQL database, CRM, or product catalog and return real answers based on real data — not guesses.
Email & calendar
The AI can read your emails, schedule meetings, send follow-ups — acting as a genuine executive assistant, not just a chat interface.
Web search
The AI can search the internet in real time, overcoming the knowledge cutoff problem for any question about current events.
Business software
Salesforce, Jobber, HubSpot, QuickBooks, Slack, Notion — the AI can read and write to the tools your business already runs on.
File systems
The AI can read documents, create files, update spreadsheets — treating your file storage as a working environment.
Custom APIs
Any service with an API can become an MCP tool. Your proprietary internal system, your booking platform, your inventory system.

MCP vs function calling — what's the difference?

Function calling is the underlying mechanism: the LLM signals "I want to call this function with these parameters," and the application executes it and returns the result. It's been around since 2023.

MCP is the standardization layer on top: a protocol that defines how tools are described, discovered, and called — so any AI can use any tool without custom integration work. Function calling is the engine; MCP is the universal adapter.

How this works in the lab product

In the contractor AI front desk we built in the lab, MCP is what connects the AI conversation to the contractor's job management software. When the AI finishes qualifying a lead, it doesn't just output text — it calls the Jobber MCP connector and creates an actual lead record. The contractor opens Jobber and the lead is already there, with all the information the AI collected.

Without MCP (or a custom API integration), the AI could only output a text summary that someone would have to copy-paste into Jobber manually. With MCP, the workflow is truly automated end to end.

The Monday signal If someone proposes an AI workflow that ends with "and then someone manually enters it into the system" — that's where you ask: "Do we have an MCP connector for that tool, or can we build one? That's the step that closes the loop."
