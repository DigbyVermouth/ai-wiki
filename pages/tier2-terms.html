Vocabulary · Tier 2

Drop these
naturally.

These terms won't make or break Monday — but weaving them in naturally signals you've gone beyond the surface. They also set you up for the week ahead.

Embeddings

Converting text into numbers (vectors) so that semantically similar text lands close together mathematically. "Best burger in town" and "top local restaurants" become nearby vectors. This is what powers semantic search and RAG. When someone says "vector search," embeddings are the mechanism.

Vector database

A database optimized to store and search embeddings. Pinecone, Chroma, Weaviate. These are the storage layer of RAG systems. When someone says "we need a knowledge base for the AI," a vector DB is almost certainly part of the architecture.

Function calling / Tool use

The mechanism by which an LLM triggers real code. The model says "I need to look up order #1234" and the system executes that query and returns the result. This is how agents do things rather than describe things. MCP is the standardized version of this.

Pipeline

A sequence of AI steps where the output of one feeds the next. Extract text from PDF → summarize → classify intent → route to department. Most production AI systems are pipelines, not single model calls.

Inference cost

The per-call cost of running an AI model. Claude Sonnet: ~$3 per million input tokens, ~$15 per million output tokens. At average business conversation lengths (~2,000 tokens), that's less than a penny per conversation. Cost only becomes a concern at very high volume or with very long context.

Evals (evaluations)

Systematic testing of AI outputs for quality, accuracy, and consistency. How you know your prompt actually works before shipping it. Running 100 test cases and scoring the results. This is what separates production-grade AI from demo-grade AI — and almost nobody in small business does it yet.

Latency

How long the AI takes to respond. For real-time customer-facing applications (live chat, voice), latency matters enormously — users notice anything over 2–3 seconds. For backend automation (nightly reports, batch processing), latency is irrelevant. Match the model to the use case.

Structured output / JSON mode

Forcing an LLM to respond in a specific format (usually JSON) instead of free-form text. Essential for automation — when the AI's output needs to be parsed by another system, you need predictable structure. Most major models support this natively now.

LangChain / LlamaIndex

Python frameworks for building RAG systems and agent pipelines in code. LangChain especially got very popular very fast — but also got a reputation for adding complexity without always adding value. Good for experienced Python teams. Usually overkill for small business automation vs. n8n or Make.

Hugging Face

The GitHub of AI models — a massive repository of open-source models, datasets, and tools. Where you go to find and download open models. If someone says "we can grab a model from HuggingFace," they mean an open-source model they can run themselves.

Reasoning models

A newer class of models (OpenAI's o1/o3, Claude's extended thinking mode) that "think" before responding — working through a problem step by step internally before outputting an answer. Much better at complex multi-step reasoning. Slower and more expensive than standard models. Right for hard problems, overkill for simple tasks.

10DLC

10-Digit Long Code — the regulatory compliance framework for business SMS in the US. If you're building anything that sends texts to customers (like the missed call text-back product), you must register your number and use case with carriers. It's a real operational requirement that trips up many AI product builders.
